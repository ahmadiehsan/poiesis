1. ترتیب کد در کامپوننت به شکل زیر باشد:
    Props
    Internal State
    Redux
    Hooks
    Handle functions
    UseEffect
    return
  2. کد یا کامپوننت مورد نظر فقط در صورتی در فولدر Framework قرار بگیرد که در باقی App ها به آن نیاز است.
  3. در import کردن از فولدر Framework همیشه path آن absolute باشد. برای مثال: framework/helpers/constants
  4. در import کردن از App داخلی همیشه path آن relative باشد. برای مثال: ../state/actions
  5. برای تمام string ها در قسمت import از " (Quotation mark) استفاده شود. برای string های به غیر از این بخش از ' (Apostrophe) استفاده شود.
  6. برای نام گذاری صفحات، Action Creator ها، Action Type ها، و Reducer از این لینک استفاده شود.
  7. فقط در صورتی که Action شامل گرفتن دیتا با API نمی‌شود می‌توان از actionTypeGenerator برای نامگذاری Action type استفاده ننمود.
  8. برای Action های دارای تعامل با API که شامل گرفتن دیتا و یا ساخت و ویرایش آن می‌شود می بایست از Loading استفاده شود.
  9. برای نشان دادن آنکه کاربر نیاز به صبر کردن دارد باید از Spinner استفاده شود.
  10. برای تمام لینک و موارد کلیک کردنی، برای حالت هاور می بایست تغییر قابل مشاهده‌ای در المان وجود داشته باشد. (مانند تغییر رنگ در آن)
  11. در زمان حذف یک مورد حتما باید warning به کاربر نمایش داده شود تا با تایید آن حذف انجام گیرد.
